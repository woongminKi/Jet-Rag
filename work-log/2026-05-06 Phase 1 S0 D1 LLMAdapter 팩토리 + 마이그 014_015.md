4. **PPTX rerouting 의 doc_id/page 미전파** — slide_idx 가 page 의미와 다름 → 의도적 NULL. 별도 `slide_idx` 컬럼 추가는 향후 검토.
5. **vision_page_cache lookup 미통합** — 마이그 015 ship 만 완료. `_enrich_pdf_with_vision` 의 lookup→miss→upsert 코드는 D2-B.
6. **모델 2.5 → 2.0 미변경** — TODO 주석으로 명시. master plan §4 정합은 D2-C 별도 commit.

### 10.2 정책 충돌 해소 (보안 경고)

sub-agent 의 **main 직접 push** 가 보안 정책 위반 경고 발생.

- CLAUDE.md = "feature 단위 commit + push" 권고 → main push 허용
- 일반 보안 정책 = "feature branch + PR review" 권고 → main push 위반

**현재 1인 개발이라 PR 리뷰 프로세스 부재** → CLAUDE.md 정책 우선이 합리적. 단, 사용자가 다른 선호 있다면 향후 commit 부터 feature branch 채택 가능.

| 옵션 | 장점 | 단점 |
|---|---|---|
| A. 현재대로 main 직접 push (CLAUDE.md 정책) | 1인 개발 효율, 빠른 ship | 회귀 시 main 오염 위험 |
| B. feature branch + 사용자 merge | 회귀 격리, 변경분 리뷰 가능 | 1인 개발에 부담, ship 속도 ↓ |
