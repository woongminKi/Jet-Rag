## 4. 트러블슈팅 (실제 만난 사례)

### 4.1 woongs.com Sedo Parking 매물

- 증상: Cloudflare Registrar 에서 $10/년 가격으로 구매 불가
- 원인: 등록자가 이미 보유 + Sedo 마켓플레이스에 호가 매물 등록
- 해결: 대안 TLD 검색 → `.com` 변형 (`woong-s.com`) 으로 회피

### 4.2 Cloudflare picker 가 다른 사이트 (`woongs-world.com`) 로 열림

- 증상: Cloudflare DNS 화면이 `woongs-world.com` 으로 열린 상태에서 record 추가 시도
- 원인: 다른 사이트가 같은 Cloudflare 계정에 함께 등록되어 있어 picker 가 다른 site 를 기본 표시
- 해결: Cancel → 좌상단 picker → `woong-s.com` 전환 → 다시 record 추가

### 4.3 Cloudflare Proxy 권장 메시지 (주황 alert)

- 증상: "Proxying is required for most security..." 경고 표시
- 원인: Cloudflare 기본 정책이 Proxy 권장
- 해결: **무시** — Vercel/Railway 가 자체 SSL 발급하므로 Proxy ON 시 SSL 충돌. **DNS only 가 정답**.

### 4.4 Railway TXT verify Content 칸 위치 불분명

- 증상: Cloudflare DNS 입력 화면에서 "Value" 칸이 없음
- 원인: Cloudflare 가 TXT 의 Value 칸을 **"Content (required)"** 로 라벨링
- 해결: Content 칸에 `railway-verify=...` 전체 값 (앞뒤 공백·따옴표 없이) 붙여넣기

### 4.5 NEXT_PUBLIC_* ENV 갱신 후 자동 redeploy 안 됨

- 증상: Vercel 의 ENV 만 바꿔도 production 에 반영 안 됨
- 원인: `NEXT_PUBLIC_*` 은 빌드 시 inlining → ENV 저장만으론 안 됨, redeploy 필수
- 해결: Deployments → 최신 deployment 의 ⋯ → Redeploy
