## 6. 사용자 수동 검수 가이드

### 6.1 위치

```bash
# 현재 작업 디렉토리 기준
open evals/results/m3_kpi8_source_match_audit_50_20260514_224631.csv
# Excel / Numbers / Google Sheets 등으로 열기
```

### 6.2 검수 절차

각 row 에 대해:

1. `query` 와 `expected_doc_title` 확인
2. `predicted_doc_id` ≠ `expected_doc_id[:8]` 면 → **0** (doc 자체 틀림)
3. 같은 doc 이면:
   - `predicted_page` 가 `relevant_pages` 또는 `acceptable_pages` 안에 있으면 → **1**
   - 인접 page (±1) 또는 `source_hint` 와 정합하면 → **1** (검수자 재량)
   - `predicted_snippet` 내용이 `query` 답에 직접 필요하면 → **1** (page label 부정확 케이스)
   - else → **0**
4. `expected_pages` 가 비어있으면 → **-1** (label 부재로 측정 불가, 통계 제외)

### 6.3 일치율 계산

```
일치율 = Σ(match_manual=1) / (n - Σ(match_manual=-1))
        게이트 ≥ 0.95
```

### 6.4 작업량 예상

- 50건 × ~1~2분 = ~1~2시간 (PRD §6.1 예상치 정합)
- 자동 채점 1=20건, 0=30건 → 0건 우선 재평가하면 더 빠름
