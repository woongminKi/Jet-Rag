### 3.5 데이터센터 p.40 false negative 분석 (G-A-008 miss 유지)

```
text_chars: 813   density: 1.62e-3   table_like: 0.00
image_area: 0.009  text_quality: 0.97  caption_score: 0.067
needs_vision: False   composite_score: 0.072
```

**원인**: PyMuPDF 가 p.40 의 표를 cell 별로 line 분리 (각 line = 단일 span, 다중 공백·탭 0). table v2 fallback (`_is_multi_column_line`) 은 `len(spans) == 1` 일 때 본문 안의 다중 공백·탭으로 column 추정 — 하지만 이미 cell 마다 별도 line 이라 single line 안에 column 신호 없음.

**해결 옵션 (D3 범위 외 — v3 sprint 후보)**:
1. **multi-line table v3** — 인접 line 들의 y 좌표 + bbox 폭 비교로 표 영역 판정 (Camelot / pdfplumber 의 ruling line 추출 idea 차용, PyMuPDF drawings)
2. **caption 임계 완화** — 0.20 → 0.05? p.40 의 caption_score 0.067 도 hit. 단, false positive 폭증 위험 (정량 측정 후 결정)
3. **page-level image-only fallback** — 본문은 다른 신호로 처리, p.40 같은 표 페이지는 별도 패턴 (예: 짧은 line 비율) 추가 도입

D3 ship 에선 **현재 false negative 유지** + 골든셋 recall 5/6 = 83.3% 동일 유지. v3 sprint 진입 시 multi-line table 휴리스틱 우선.
