# 2026-05-04 — W25 D5: doc 페이지 매칭 청크 표시 (`?q=` 본격 처리)

## 배경

W25 D3 Phase 1 으로 검색 결과 카드의 `+89개 더 매칭 (이 문서에서 모두 보기 →)` 링크는
`/doc/{id}?q=...` 로 이동만 했다 — doc 페이지가 `?q=` 자체를 무시.

사용자 의도는 명확:

- 결과 카드에 표시된 매칭 청크 3개 + 안 보이는 89개 = **총 92개 매칭 청크 모두 보기**.
- 현재는 doc 페이지에서 단순 메타 (요약 / 태그 / 청크 99개 / 처리 완료) 만 표시.

senior-qa D2 후속 큐 P1 — **D5 본격 처리**.

## 진단 (구현 전)

- `web/src/app/doc/[id]/page.tsx` — `useSearchParams().get('uploaded')` 만 처리, `q` 무시.
- 백엔드 `api/app/routers/search.py` — `?doc_id=` query param 으로 doc 스코프 검색 지원 (W11 D4).
- 핵심 결함: `_MAX_MATCHED_CHUNKS_PER_DOC=3` cap 이 doc_id 스코프에서도 적용 → 응답에 본문 3개만.
  - 단, `matched_chunk_count` 는 unique 수 (90+개) 정확. 본문 cap 만 문제.

### doc 스코프 검색 fetch (q 있을 때만)

React 19 패턴 (AGENTS.md 패턴 2): useEffect 안 동기 setState 금지 → `.then/.catch` 콜백 안 setState 만 사용.

```tsx
useEffect(() => {
  if (!q) return;
  let cancelled = false;
  searchDocuments(q, 100, 0, docId, 'hybrid')
    .then((resp) => {
      if (cancelled) return;
      const hit = resp.items[0];
      setMatchedChunks(hit ? hit.matched_chunks : []);
    })
    .catch(() => { if (!cancelled) setMatchedChunks([]); });
  return () => { cancelled = true; };
}, [docId, q]);
```

### 옵션 평가 (사용자 권고 vs 최종)

- **옵션 (a)** — 작은 텍스트 `매칭 강도 99%` 단독: 채택.
- **옵션 (b)** — `RRF 0.0161` 위치에 `99%` 만: 라벨 제거 시 `매칭 강도` 와 의미 단절 → 부분 채택 (위치 b, 텍스트 a).
- **옵션 (c)** — 라벨 + ⓘ + % + 막대 (RelevanceLabel 그대로): 미채택. 92행 × 막대 = 시각 노이즈.

→ 최종: 위치는 (b), 텍스트는 `매칭 강도 99%`. 막대 없음. ⓘ 는 헤더에 1번만.

// chunks[0].rrf_score = max (백엔드 score desc 정렬 보장).
const maxRrf =
  typeof chunks[0]?.rrf_score === 'number' && chunks[0].rrf_score > 0
    ? chunks[0].rrf_score
