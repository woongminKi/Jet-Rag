## 0. 핵심 요약

수익화 sprint 의 **W3 = "Free/Pro 플랜 한도를 DB 로 정의하고 로그인 사용자의 답변·업로드에 402 quota 를 W2 rate limit 게이트에 통합 적용"** 을 구현·배포했다.

**적용된 정책 (사용자 확정 2026-07-06):**
- **플랜 = DB seed**: `plans(code PK)` — free(문서 10 · 답변 일 5회 · 0원) / pro(200 · 50회 · 6,900원 가안). `UPDATE plans` 만으로 조정 — 재배포 불필요
- **구독 = `subscriptions(user_id PK)`**: status active/past_due 만 유효 플랜 (past_due = W5-6 grace 예약). canceled/행 없음 = free. billing_key 는 W5-6 카카오페이 SID 예약
- **통합 게이트**: `enforce_rate_limit` 확장 — usage_counters increment **1회** 결과로 ① 플랜 quota(402, 로그인 유저만) → ② abuse cap(429, 전체) 순차 판정. 별도 dependency 였으면 카운터 +2 버그
- **문서 한도 = 보유 문서 수** (`deleted_at IS NULL` COUNT ≥ max_documents → 402). 삭제하면 재업로드 가능. W2 일일 docs rate limit(남용 방어)과 역할 분리
- **OWNER bypass**: `user_id == owner_user_id` 면 quota skip (abuse cap 은 유지). 익명 데모는 플랜 없음 — 429 만
- **fail-open**: 플랜 조회/문서 카운트 실패 시 quota skip + warning. 회복 토글 `JETRAG_QUOTA_ENFORCEMENT_ENABLED`(기본 true)
- **W2 동작 변화 1건**: quota 대상이면 abuse cap=0 이어도 카운터 increment (기존 테스트 1건 `test_unlimited_when_cap_zero_and_quota_off` 로 교체)

**실행 방식**: superpowers:subagent-driven-development — Task 별 fresh 서브에이전트 + 2단계 리뷰(spec → quality). 최종 홀리스틱 리뷰(opus) verdict: **READY WITH NOTES** (blocker 0).

### 1.2 DB (마이그 022 — production 적용 완료)

- `plans(code TEXT PK, name, max_documents, answers_per_day, price_krw, created_at)` + seed free/pro. 0 이하 = 무제한
- `subscriptions(user_id UUID PK, plan_code FK, status CHECK(active/past_due/canceled) DEFAULT active, current_period_end, billing_key, created_at, updated_at)`
- RLS: plans = 공개 SELECT(가격표) + service_role ALL / subscriptions = 본인 SELECT(019 패턴) + service_role ALL
- 멱등 (IF NOT EXISTS / ON CONFLICT DO NOTHING / DROP POLICY IF EXISTS)

### 1.3 백엔드

- **`api/app/config.py`**: `quota_enforcement_enabled: bool = True` + `_parse_bool("JETRAG_QUOTA_ENFORCEMENT_ENABLED", True)`
- **`api/app/services/quota.py`** (기존 파일에 병합 — 구 `is_quota_exhausted` LLM 에러 분류기와 공존, 분리는 이연):
  - `PlanLimits(code, max_documents, answers_per_day)` frozen dataclass
  - `get_effective_plan(user_id) -> PlanLimits|None` — 구독 없음/canceled→free, active/past_due→plan_code, 실패→None(fail-open)
  - `count_active_documents(user_id) -> int|None` — `count="exact"` + `limit(1)`(payload 최소화, 주석 명시)
  - `get_todays_count(user_key, metric) -> int` — 표시 전용 (docstring 에 "enforcement 사용 금지" 경고)
- **`api/app/services/rate_limit.py`**: `enforce_rate_limit` 통합 게이트 — `quota_active = 토글 && is_authenticated && user_id != owner`, increment 1회 → 402(answers `count > cap` / docs 보유수 `>= max`) → 429. 익명은 `get_effective_plan` 호출 자체가 없음
- **`api/app/routers/me.py`** (신규): `GET /me/plan` — 라우터 레벨 `require_authenticated_user`. 플랜 None → 503. 응답 = plan_code/max_documents/answers_per_day/answers_used_today/documents_count. user_key 정합 확인됨 (게이트와 동일 키)
