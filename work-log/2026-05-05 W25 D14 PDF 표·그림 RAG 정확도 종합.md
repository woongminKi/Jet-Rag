## 1. 진단 (직접 검증)

### 1.1 PyMuPDF 의 PDF 처리 본질적 한계

| 항목 | 현황 | 의미 |
|---|---|---|
| 표 추출 | `find_tables()` API 미사용, raw text 만 추출 | 표 cell 순서 뒤섞임 / 일부 누락 |
| PDF 안 이미지 | block type=1 (이미지) 을 if 문에서 완전 무시 | 그림 정보 chunks 0건 |
| 스캔 PDF | text<50자 일 때만 vision 호출 (이미 ship) | 일반 텍스트 PDF 의 그림은 무시 |
| Page→PNG | `get_pixmap(dpi=150).tobytes('png')` (스캔 fallback) | 인프라는 이미 존재 |
| Vision adapter | `GeminiVisionCaptioner` 어댑터 ship 완료 | 호출 인프라 존재 |

→ **결론**: vision 인프라는 이미 있지만 일반 PDF 에 적용 안 됨이 문제의 본질.

### 1.2 본 PDF 의 chunks 분포 (DB 직접 조회)

```

  chunk 16 (5자):  "억 5~"
  chunk 17 (14자): "억 25.33 개 이상 2"
  → 표 cell 정보가 4~5 chunks 로 흩어짐
```
