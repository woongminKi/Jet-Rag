## 2. 내일 새 macOS 컴퓨터에서 이어가는 절차

### 2.1 사전 설치 확인 (이미 있으면 건너뜀)

| 도구 | 설치 명령 | 버전 확인 |
|---|---|---|
| Homebrew | (macOS 공식 스크립트) | `brew --version` |
| Git | `brew install git` | `git --version` |
| GitHub CLI | `brew install gh` | `gh --version` |
| **uv** (Python 관리자) | `brew install uv` | `uv --version` |
| Python 3.12 | `uv python install 3.12` | `python --version` |
| Node.js 20+ | `brew install node` | `node --version` |
| **pnpm** | `npm install -g pnpm` | `pnpm --version` |
| Cursor 또는 VS Code | `brew install --cask cursor` | — |

### 2.2 GitHub CLI 로그인 (개인 계정)

```bash
gh auth login
# GitHub.com → HTTPS → 브라우저 → 개인 계정(woongminKi)
gh auth status
```

### 2.3 repo clone 및 작업 디렉터리 세팅

```bash
mkdir -p ~/projects && cd ~/projects
git clone https://github.com/woongminKi/Jet-Lag.git
cd Jet-Lag
```

### 2.4 repo-local git 설정 (**clone 직후 필수**)

회사 계정이 global에 잡혀 있을 가능성이 높으므로 repo 단위로 명시:

```bash
git config --local user.email "dndals1991@gmail.com"
git config --local user.name  "woongminKi"

git config --get user.email  # dndals1991@gmail.com 확인
```

### 2.5 `.env` 복원 (사용자가 별도 안전 경로로 옮겨온 값)

```bash
cp .env.example .env
# 에디터로 .env 열어 세 값 채우기:
#   SUPABASE_URL=...
#   SUPABASE_KEY=eyJ...
#   GEMINI_API_KEY=AIzaSy...
```

`.env` 파일은 `.gitignore`로 차단되어 있으니 절대 커밋되지 않음. 실수 방지를 위해 커밋 전 `git status`에 `.env`가 나오지 않는지 한 번 더 확인.

## 8. 문제 해결 FAQ

### Q. 새 맥에 uv·pnpm·gh 없음
→ §2.1 표대로 순차 설치. Homebrew 먼저.

### Q. `git push` 시 회사 이메일로 찍힘
→ §2.4 놓친 것. `git config --local user.email dndals1991@gmail.com` 재실행. 이미 잘못 찍힌 커밋은 `git commit --amend --reset-author` 후 `git push --force-with-lease`.

### Q. Supabase pgvector 어떻게 켜나
→ 대시보드 → Database → Extensions → `vector` 검색 → Enable. Day 2 첫 작업에 포함 예정.

### Q. Gemini 무료 한도 초과
→ 기획서 §9.9 / §D-7-3 참조. 이미지 해상도 1024px 다운스케일 + 일 1,500회 한도 내 작업.

### Q. 오늘 집 컴퓨터에서 쓰던 `Jet-Lag/` 폴더는 어쩌나
→ `~/projects/Jet-Lag/` 아래에 clone하면 집 컴퓨터와 새 맥 모두 같은 리모트를 바라보는 클론. 양쪽 모두 `git pull` 습관화. 충돌 예상되면 한쪽에서 작업 전 `git status` 확인.
