## 1. 비판적 재검토

### 1.1 통합 vs 분리 테이블

| 옵션 | 설계 | 결정 |
|---|---|---|
| 통합 `metrics_log` + type 컬럼 | 단일 테이블 | ❌ schema 다름 (Vision 은 success/quota/source_type, Search 는 took_ms/mode/dense_hits…) — 컬럼 NULL 분산 |
| **분리 `vision_usage_log` / `search_metrics_log`** | 각자 정확한 schema + 인덱스 | ✅ 채택 |

### 1.2 query_text 저장 vs hash

- 단일 사용자 MVP — 자연어 쿼리 PII 위험 0 (사용자 본인)
- 멀티 사용자 도입 시 query_hash (MD5/SHA256) 변경 검토 (DE-21 PII 정책 일관)
- 본 마이그레이션은 plain text 채택 + DE 결정 후 변경 가능

### 1.3 RLS 정책

- service_role only (anon/authenticated 차단) — 005·006 동일
- 단일 사용자 MVP user_id 컬럼 미도입 (W2 §3.M 정책 일관)

### 1.4 인덱스 정책

| 인덱스 | 용도 |
|---|---|
| PK serial | 단조 증가 |
| `*_recorded_at DESC` | 최근 N건 / 시간 범위 |
| `vision quota partial` | quota_exhausted=TRUE 만 (대부분 row 는 FALSE) |
| `search mode composite` | mode 별 ablation 비교 |
| `search fallback partial` | fallback_reason IS NOT NULL 만 |

→ partial index 로 disk 절약 + 핫 path 빠른 조회.

### 2.2 vision_usage_log schema

```sql
CREATE TABLE vision_usage_log (
    call_id          BIGSERIAL PRIMARY KEY,
    called_at        TIMESTAMPTZ NOT NULL DEFAULT now(),
    success          BOOLEAN NOT NULL,
    error_msg        TEXT,
    quota_exhausted  BOOLEAN NOT NULL DEFAULT FALSE,
    source_type      TEXT  -- 'image' / 'pdf_scan' / 'pptx_rerouting' / 'pptx_augment'
);
```

### 2.3 search_metrics_log schema

```sql
CREATE TABLE search_metrics_log (
    record_id          BIGSERIAL PRIMARY KEY,
    recorded_at        TIMESTAMPTZ NOT NULL DEFAULT now(),
    took_ms            INT NOT NULL,
    dense_hits         INT NOT NULL DEFAULT 0,
    sparse_hits        INT NOT NULL DEFAULT 0,
    fused              INT NOT NULL DEFAULT 0,
    has_dense          BOOLEAN NOT NULL,
    fallback_reason    TEXT,
    embed_cache_hit    BOOLEAN NOT NULL DEFAULT FALSE,
    mode               TEXT NOT NULL DEFAULT 'hybrid'
                       CHECK (mode IN ('hybrid', 'dense', 'sparse')),
    query_text         TEXT
);
```
