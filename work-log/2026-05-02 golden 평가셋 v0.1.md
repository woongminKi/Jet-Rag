## 4. 측정 가이드

### 4.1 Recall@10 정성 sanity check (Day 6, 정량은 W6+)

각 query 당 다음 3 항목 측정:

1. **top-3 hit**: top-3 검색 결과 안에 `expected_doc_ids` 중 최소 1건 포함되었는가 (Y/N)
2. **top-10 hit**: top-10 안에 포함되었는가 (Y/N) — Recall@10 의 정성 proxy
3. **expected_keywords 등장**: top-3 chunk 텍스트에 `expected_keywords` 중 최소 1개 등장하는가 (Y/N) — 출처 일치율 §13.1 정성 proxy

### 4.2 KPI 매핑

| KPI (§13.1) | 측정 방법 |
|---|---|
| Context Recall@10 (목표 0.90, W6+) | 20건 평균 top-10 hit 비율. v0.1 base 는 N=20 으로 통계적 power 부족 — Day 6 는 정성 sanity. |
| 출처 일치율 (목표 0.95) | 20건 평균 expected_keywords 등장 비율. 모범 경로. |
| 하이브리드 우세 (+5pp, AC 폐기, hooks 만 ship) | dense_only / hybrid 두 경로 hooks 호출 후 결과 비교 — search_metrics ring buffer 활용. |
| HWPX section_title ≥ 30% | G-001~005, G-011~012, G-016~017 의 9건 (HWPX 매칭) 결과 chunk 의 section_title 채움 비율. F (HWPX heading) ship 후 측정. |
| P95 검색 응답 < 500ms | 20건 took_ms 평균·p95. cold start 1~2초 만 예외 — embedding cache (W4-Q-3) 후 재측정. |

### 4.3 정성 검토 체크리스트 (Day 6 사용자용)

- [ ] 자연어 10건: top-3 안에 expected_doc 들어오는 건수 (목표 7건 이상)
- [ ] 키워드 5건: 4건 이상 top-1 hit (sparse 강점)
- [ ] 메타 혼합 5건: 5건 모두 top-3 hit (메타 필터 회귀)
- [ ] 한국어 어미 변형 강건성 (G-003 `되더라`) — sparse 가 0 이어도 dense 가 보완
- [ ] 영어+한국어 혼합 (G-019 `AI 투자`) — 두 언어 모두 토큰화
- [ ] 숫자+특수문자 (G-015 `2.2%`) — TokenBigram 의 숫자 처리
- [ ] Top-3 안에 페르소나 A 가 "이게 정답이네" 라고 인정할 chunk 가 있는가 (정성)
- [ ] "다른 doc 의 chunk 가 잘못 섞여 있다" 사례가 있는가 (false positive)
