## 4. 남은 이슈

### Q-S3-D5-1. 실 BGE-reranker 환경에서 재측정 필요 (블로킹)

`--mock-reranker` 사용 — deterministic chunk_id sha1 기반 score 라 의미 매칭은
흉내내지 못함. cache hit / degrade / MMR / latency / 코드 path 측정 의미는 보존.

**관찰**: combo b/c 의 R@10=0.475 / top-1=0.239 vs combo a 의 R@10=0.717 /
top-1=0.708 — mock 의 random score 가 RRF score 를 덮어쓰면서 의도적 reranker
효과 검증 불가능. 결과 자체는 mock 가설 부합 (random reranker 가 baseline 보다
악화) — 코드 path 정상 작동 검증으로는 의미 충분.

해소: 실 BGE-reranker 환경에서 본 스크립트 `--mock-reranker` 옵션 제거 후
재실행. HF API 무료 cap 한도 (월 1000회 default) 내 1회 측정 가능.

### Q-S3-D5-2. paid_decomposition (D3 ENV ON) 회귀 측정 미진행

`JETRAG_PAID_DECOMPOSITION_ENABLED=true` 검증은 본 D5 범위 외 — 사용자 명시 승인
후만 별도 phase (S3 D5-ext) 로 진행. Flash-Lite 호출당 비용 ~$0.001 × 157 = $0.16
무료 티어 잔량 체크 필수.

해소: D5-ext sprint 진입 시 budget_guard 잔량 확인 + ENV ON + 본 스크립트
재실행 (옵션 추가 필요 시 본 도구에 `--paid-decomp` 토글 추가 검토).

### Q-S3-D5-3. doc 매칭 fail 11건 정정 + cross-doc chunk-labeled row 보강

  비어있어 chunk-level 평가 0 — MMR 의 cross-doc 효과 측정 자체가 현재 골든셋
  으로는 불가.

   → golden_v1.csv 의 해당 row title/doc_id 보강.

### Q-S3-D5-4. combo a 의 P95 66s — HF read timeout retry 누적

stderr 로그에서 9건의 `bge-m3.embed_query transient 실패` 관찰. 5~10s 후 retry
3회 → 1 row 에 누적 30~60s. 운영 환경에서는 같은 timeout 가 반복적으로 발생하면
SLO §10.11 "URL <30초" 와도 충돌.

원인 가설:
- (a) HF 서버 cold start (Inference API 무료 티어 idle 후 재가동 시 5~20s 지연 정상)
- (b) network 일시 단절 (combo a 측정 시간 1517s 동안 독점 통신)
- (c) BGE-M3 모델 자체 inference 시간 (1024 dim) — 가능성 낮음

해소: combo b/c 시점에는 cache warm + retry 0 → HF API 안정성 가설 (a) 가
가장 유력. 운영 시작 전 dummy warmup query 1회 발사 권고 (cold→warm 전환).
