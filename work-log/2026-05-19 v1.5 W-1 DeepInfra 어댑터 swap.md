## 4. senior-developer 의 설계 결정 사항

1. **반환 타입 힌트 변경** — `get_bgem3_provider()` 무명 반환. Protocol 구조적 타이핑이라 호출 사이트 안전 (`dense_dim`/`embed`/`embed_batch`/`embed_query`/`_last_cache_*` attribute 만 사용).
2. **unknown ENV graceful fallback** — `JETRAG_EMBED_PROVIDER=openai-bge` 같은 오타 → startup crash 대신 default `hf` + warning. 운영 안전성 우선.
3. **DeepInfra batch `index` 정렬 방어** — OpenAI spec 상 입력 순서 보장되지만 `sorted(items, key=lambda d: int(d.get("index", 0)))` 로 spec drift 방어.
4. **transient 분류 alias 두 개 공존** — `is_transient_hf_error` (HF) + `is_transient_deepinfra_error` (DeepInfra). 미래에 어댑터별 다른 fallback 전략 필요 시 분기.
5. **영구 캐시 `_MODEL_ID` 통일** — `BAAI/bge-m3` 단일 키 → HF↔DeepInfra 가 같은 entry 공유 (W-0 cosine 0.999984 PASS 로 안전).
6. **lru_cache 누수 방지** — `get_bgem3_provider.cache_clear()` 를 factory 테스트 setUp/tearDown 마다 호출. 운영 코드는 freeze 의도 보존.
7. **helper 복제** — `_with_retry`/`_parse_retry_after`/`_is_retryable` 어댑터 안에 복사 (reranker 패턴). 어댑터별 독립 진화 보장.

## 5. 사이드 이펙트

### 5.1 HF default 경로 100% 보존

ENV 미설정 시 기존 코드 path 와 동일. lazy import 도 그대로 보존 — HF-only 환경에서 DeepInfra 모듈 로드 안 함 (`token` 없는 단위 테스트 보호).

### 5.2 영구 캐시 entry 공유

- `embed_query_cache._MODEL_ID = "BAAI/bge-m3"` 단일 정의
- `_cache_key()` 알고리즘 100% 동일 (sha256(NFC(strip(text))))
- → 같은 query 텍스트는 같은 cache row → swap 시 의도하지 않은 결과 0
- First-writer wins: HF 가 먼저 write 한 entry 는 DeepInfra 가 read 해서 그대로 반환 (반대도 동일). W-0 cosine 0.999984 PASS 가 추가 보증.

### 5.3 운영 ENV 변경 시 새 프로세스 필요

`lru_cache(maxsize=1)` freeze 는 의도된 동작:
- 프로세스당 1개 `httpx.Client` 공유 → FD leak 방지
- 런타임 ENV 변경 가능성 0 (재시작이 의도)

운영 시 Railway 에서 `JETRAG_EMBED_PROVIDER=deepinfra` 등록 + redeploy 1회면 swap 완료.
