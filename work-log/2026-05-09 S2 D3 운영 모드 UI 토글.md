### Backend 우선순위 정책 (Q-S2-1e A 안 채택)

`_resolve_page_cap_for_doc` 의 단일 진실원:

1. `settings.vision_page_cap_per_doc <= 0` → **0 (무한, 전역 kill switch)** — mode/override 전부 무시
2. 그 외 → `override if override is not None else settings.vision_page_cap_per_doc`

→ 운영자가 ENV 로 page cap 자체를 끄고 싶을 때 (cost cap 만으로 제어) 회복 토글 보장.
