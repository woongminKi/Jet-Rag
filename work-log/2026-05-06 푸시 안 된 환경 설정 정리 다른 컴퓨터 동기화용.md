## 4. `.codex/` 디렉토리 (codex CLI 도구 설정)

### 4.1 무엇인가

OpenAI Codex CLI (또는 codex 도구) 가 본 프로젝트에서 사용하는 설정 + 에이전트 정의. **시크릿 토큰 포함**.

### 4.2 본 컴퓨터의 구조

```
.codex/
├── config.toml           # MCP 서버 설정 + Supabase Personal Access Token (sbp_...)
└── agents/               # codex 에이전트 정의 (디렉토리)
```

### 4.3 `.codex/config.toml` 의 시크릿

```toml
[mcp_servers.supabase-jetrag]
args = ["-y", "@supabase/mcp-server-supabase@latest", "--read-only", "--project-ref=mpmtydudhojpukuuadrd"]
command = "npx"

[mcp_servers.supabase-jetrag.env]
SUPABASE_ACCESS_TOKEN = "sbp_..."   # ⚠ 평문 저장 — git 절대 금지
```

`SUPABASE_ACCESS_TOKEN` 은 **Supabase 계정의 Personal Access Token**.
- 출처: https://supabase.com/dashboard/account/tokens
- 권한: 너의 모든 Supabase 프로젝트 접근 (계정 단위)
- ⚠ 노출 시 즉시 회전 권고

### 4.4 본 commit 의 `.gitignore` 변경

```diff
 .mcp.json
+.codex/
```

→ 다음에 실수로 `.codex/` 가 git 에 들어가는 것 방지.

### 4.5 다른 컴퓨터에서 codex CLI 사용 시

본 컴퓨터의 `.codex/` 를 **그대로 복사 X** — 시크릿 포함이라 동기화 채널이 안전해야 함. 권고 순서:

1. **codex CLI 설치** (별도 가이드 필요 시 codex 공식 문서)
2. **새 컴퓨터에서 codex init 으로 새 `.codex/config.toml` 생성**
3. **Supabase Personal Access Token 새로 발급** (가능하면 회전, 같은 토큰 재사용 자제)
4. **에이전트 정의 (`agents/`) 만 본 컴퓨터에서 복사** — 시크릿 없는 부분

또는 codex 가 필요 없다면 `.codex/` 동기화 자체 skip.

## 7. 시크릿 동기화 권고 (보안)

| 방식 | 장점 | 단점 |
|---|---|---|
| **1Password / Bitwarden / KeePass** | 양 컴퓨터 자동 동기화, 회전 시 한 곳만 갱신, 감사 로그 | 도구 설치 필요 |
| **iCloud Keychain (Mac 전용)** | 자동, 무료 | 안드로이드·Linux 미지원 |
| **USB 직접 복사** | 도구 무관, 노출 채널 통제 | 수동, 컴퓨터 늘면 동기화 부담 |
| **AWS Secrets Manager / Doppler** | 팀 협업 + 자동 동기화 + 감사 | 비용 + infra 복잡도 |

**현재 단일 사용자 + 2대 컴퓨터** 패턴이라 **1Password 또는 iCloud Keychain 권고**. work-log 본 문서를 secret manager 의 메모 영역에 같이 저장해두면 어떤 변수가 필요한지 즉시 파악 가능.
