## W-14 `ingest_jobs` orphaned `running` watchdog

**범위**: $0, 코드 변경만 (paid 호출 0). PRD §5 M0-a W-14. BackgroundTasks 인제스트 도중 프로세스가 비정상 종료하면 `ingest_jobs` 가 `status='running'` 인 채로 영구히 남는 문제(2026-05-12 sample-report 사고 — `cd912e70…` job 이 `current_stage=embed` 로 stale)를 보수적으로 정리한다.

### 설계 선택

- **선택 A — chunks 절대 미정리**: watchdog 은 `ingest_jobs.status` 만 `failed` 로 마킹. embed 도중 멈춘 job 의 부분 chunk 를 보존한다 (sample-report 교훈 — `_cleanup_failed_doc` 류 호출은 1000 chunk 를 날린다). 복구는 `evals/_repair_*` 도구 몫. `error_msg` 에 `"watchdog: orphaned running > Nh (stage=...) — 프로세스 비정상 종료 추정, status만 마킹 (chunks 미정리)"` 명시.
- **선택 B — `started_at IS NULL` 인 running 은 미대상**: `start_job` 이 항상 `started_at` 을 set 하므로 정상 흐름에서 발생하지 않는 케이스 + cutoff 비교가 불가능. `.lt("started_at", cutoff)` 필터가 NULL row 를 자연히 제외해 별도 분기 불요.
- **threshold**: config 기본 24h. SLO §10.11 최장(이미지PDF20+ <3분) 대비 압도적 margin.
- **UPDATE 멱등**: `UPDATE ... WHERE id=... AND status='running'` — startup hook + 수동 CLI 동시 실행 시 이미 terminal 로 전이한 row 는 0 rows affected.
