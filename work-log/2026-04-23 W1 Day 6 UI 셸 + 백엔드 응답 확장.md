**핵심 변경**
- `/search` 응답 구조 전환: 1행=1청크 평탄 → **doc 단위 그룹화**. v0 카드 1장 = doc 1개로 자연스럽게 매핑됨. `total` = 매칭 doc 수, `matched_chunk_count` = 한 doc 안의 진짜 매칭 청크 수.
- `relevance` 휴리스틱 산식 (W3 RRF 도입 전 임시): `min(1.0, 0.5 * matched/total + 0.3 * title_hit + 0.2 * tag_hit)`. 산식만 W3 에서 교체하면 스키마는 그대로 유지 가능하도록 `_compute_relevance` 함수 분리.
- `matched_chunks[].highlight: list[list[int]]` — 잘린 스니펫 내부에서 매칭 위치 (start, end) 인덱스 리스트. 프론트가 `<mark>` 래핑.
- `/stats.popular_tags` — Python `Counter` 로 메모리 집계 (이미 docs 전체를 들고 있어 추가 비용 0). `added_this_month`, `added_last_7d` 는 KST 기준.
- CORS 는 FastAPI 내장 `CORSMiddleware`. **외부 패키지 추가 0**.

| F3 | 중간 | 폴링 catch 블록에 시간 제한 없음 → 백엔드 다운 시 1.5초 무한 폭주 | 연속 실패 5회 또는 5분 타임아웃 시 `timedOut=true` 로 중단 + 로컬 backoff (커밋 11) |
