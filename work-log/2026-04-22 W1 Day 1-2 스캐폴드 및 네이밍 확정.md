## 7. 내일(또는 새 맥에서) 시작점 — W1 Day 3 스코프

핸드오프 가이드(`2026-04-22 작업 이어가기 가이드.md`) §2 절차를 따르면 세팅은 5분 안에 끝난다. Day 3 본 작업은:

1. **Supabase 스키마 설계** — `senior-developer` 호출하여 `documents` / `chunks` / `ingest_jobs` / `ingest_logs` + 인덱스 + RLS 정책을 마이그레이션 SQL 한 장으로 받기
2. `api/migrations/001_init.sql` 작성 → Supabase SQL Editor에서 실행
3. Storage bucket `documents` 생성(Private) 및 `BlobStorage`의 Supabase 구현체
4. `VectorStore`의 Supabase/pgvector 구현체 (upsert_chunks, search_dense, search_sparse)
5. `POST /documents` 업로드 엔드포인트 + FastAPI `BackgroundTasks` 큐 뼈대 — 동기 응답 < 2초(§10.11 SLO), `ingest_jobs` 레코드 생성
6. SHA-256 기반 Tier 1 dedup

Day 4~5에서 포맷별 추출(PDF) → 청킹 → 임베딩 → 적재 → 간단 키워드 검색 API로 이어진다. W1 전체 목표는 **"PDF 1개 업로드 → 키워드 검색 가능"**.
