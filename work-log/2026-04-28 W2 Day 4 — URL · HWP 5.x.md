## 1. 오늘 달성 (Day 4)

### 1.1 항목 E — `UrlParser` + `POST /documents/url` 엔드포인트
- `app/adapters/impl/url_parser.py` 신설. `trafilatura.extract` (favor_precision=True) 로 본문 추출, 빈 본문 시 raise
- `app/routers/_url_gate.py` 신설. 12 종 SSRF 케이스 검증 (스킴 + 명시 호스트 + private/loopback/link-local IP)
- `POST /documents/url` (JSON body: `url`, optional `title`/`source_channel`):
  - SSRF 검증 → httpx.AsyncClient (timeout 10s, follow_redirects, UA) → HTML bytes
  - SHA-256 dedup + Tier 1 동일 흐름
  - title 자동 추정: payload → trafilatura.extract_metadata.title → URL hostname
  - `flags.source_url` 에 원본 URL 보존 (스키마 변경 없이)
  - 기존 `run_full_ingest` 재사용 (Storage upload + storage_path update + pipeline)
- end-to-end smoke (en.wikipedia.org/wiki/RAG):
  - 수신 1257ms ✓ (< 2초)
  - 파이프라인 17초 완주 (chunks 36 · tags 15 · 한국어 자연 요약)
  - title 자동 추출: "Retrieval-augmented generation - Wikipedia"

### 1.2 항목 F — `Hwp5Parser` (pyhwp 기반 HWP 5.x)
- `app/adapters/impl/hwp_parser.py` 신설. **pyhwp Python API 가 매우 low-level → `hwp5txt` CLI subprocess 패턴** (DE-40)
- venv bin 의 `hwp5txt` 를 `Path(sys.executable).parent / "hwp5txt"` 로 portable 호출
- 임시 파일 → CLI → stdout 텍스트 → 단락 분할 (\\n\\n 우선)
- timeout 30s, 실패 시 stderr 메시지 그대로 노출 (사용자가 원인 인지 가능)
- unit smoke: CLI 경로 ✓ · can_parse 분기 ✓ · invalid/empty bytes → "Not an OLE2 Compound Binary File" 정상 raise ✓
- 실제 HWP 5.x 샘플은 사용자 제공 시 정식 smoke 진행 (KPI 측정 단계와 묶음)

## 2. 결정

| # | 결정 | 근거 |
|---|---|---|
| **DE-40 (Day 4)** | `Hwp5Parser` 는 `hwp5txt` CLI subprocess 호출 (vs pyhwp Python API 직접) | pyhwp Python API (Hwp5File/xmlmodel) 가 매우 low-level (xml events·BSTR·CompoundType 등). CLI 가 안정적·이식적·관리 비용 낮음. portability 는 `Path(sys.executable).parent / "hwp5txt"` 로 확보 |
| **DE-41 (Day 4)** | URL 의 원본은 `flags.source_url` 에 저장 (vs 새 컬럼) | 스키마 변경 없이 충분. W4 `/doc/[id]` 출처 표시 시 그대로 재활용 가능 |
| **DE-42 (Day 4)** | URL fetch 의 SSRF 차단 = `loopback / private / link-local / reserved / multicast` IP | 169.254 메타데이터 + 10/8 + 192.168/16 + 172.16/12 + 127/8 + multicast/reserved. RFC 6890 표준 patterns. 12/12 단위 케이스 통과 |
| **DE-43 (Day 4)** | URL title 우선순위 = payload > trafilatura.extract_metadata.title > URL hostname | 사용자 명시 의도 우선, 자동 추출이 강하면 hostname fallback 거의 도달 X |
