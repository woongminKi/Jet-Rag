## 5. `.env` 복원 — 5 개 필수 키

`.env.example` 를 `.env` 로 복사 후 값 채우기:

```bash
cp .env.example .env
# 편집기로 .env 열고 아래 5개 값 입력
```

| 키 | 어디서 얻나 | 비고 |
|---|---|---|
| `SUPABASE_URL` | Supabase 대시보드 → Settings → Data API | `https://mpmtydudhojpukuuadrd.supabase.co` |
| `SUPABASE_KEY` | Settings → API Keys → anon (public) | 프론트엔드 공용 (아직 미사용) |
| `SUPABASE_SERVICE_ROLE_KEY` | Settings → API Keys → service_role (secret, Reveal) | **백엔드 전용, 절대 공개 금지** |
| `GEMINI_API_KEY` | https://aistudio.google.com/app/apikey | 모델 `gemini-2.5-flash` 사용 중 (2.0 은 유료 전환) |
| `HF_API_TOKEN` | https://huggingface.co/settings/tokens → New token (Read) | BGE-M3 임베딩용 |

선택 키 (없으면 config 기본값 사용):
- `SUPABASE_STORAGE_BUCKET` (기본 `documents`)
- `DEFAULT_USER_ID` (기본 `00000000-0000-0000-0000-000000000001`)

### 이동 방법 (사용자 개인 판단)

PWM (1Password / Bitwarden) · AirDrop · iCloud Drive 임시 공유 등. 이 문서에는 **실제 키 값은 적지 않음** (문서가 공개 repo 에 커밋됨).

### 이동 후 검증
```bash
grep -c "^[A-Z_]\+=" .env    # 5개 이상이면 OK
git status                    # .env 가 Untracked/Modified 에 안 떠야 정상 (ignored)
```

## 6. Python 백엔드 셋업

```bash
cd api
uv sync                                   # pyproject.toml / uv.lock 기준 의존성 설치
uv run uvicorn app.main:app --reload      # 서버 기동 (기본 포트 8000)
```

### 브라우저 동작 확인

1. `http://127.0.0.1:8000` → 자동으로 `/docs` 리다이렉트
2. Swagger UI 에서 `GET /health` → `{"status": "ok"}` (외부 서비스 안 탐, 인증 불필요)
3. `GET /stats` Execute → Supabase 연결 + service_role 키 유효 확인 (예상: `documents.total=4`, `chunks_total=378`)
4. `GET /search?q=반도체` Execute → total 70+ (기존 데이터 유지)

위 4 단계가 모두 성공하면 백엔드 동작 100% 복원.

### 실패 시 체크포인트

| 증상 | 원인 | 조치 |
|---|---|---|
| `/health` 조차 안 뜸 | 서버 기동 실패 | 콘솔 로그 확인. `uv sync` 재실행 |
| `/stats` 500 error | Supabase 키 오입력 | `.env` 의 `SUPABASE_URL`·`SUPABASE_SERVICE_ROLE_KEY` 재확인 |
| `/search` 결과 0 건 | Supabase 가 비어있음 | 정상. 새 PDF 업로드하면 채워짐 |
| `POST /documents` 업로드 후 `tag_summarize` fail | Gemini 키 문제 또는 RPD 한도 (20/일) | `.env` 의 `GEMINI_API_KEY` 재확인. 한도 초과면 graceful 하게 태그 NULL 로 저장됨 (파이프라인 계속) |

## 12. FAQ

### Q. 새 머신에서 `uv sync` 가 `pymupdf` 설치 시 매우 느림
→ 정상. 22MB 다운로드. 이후부터 캐시.

### Q. Gemini 호출이 `429 RESOURCE_EXHAUSTED`
→ RPD 20 초과. 24시간 기다리거나 결제 프로파일 연결. 또는 `gemini_llm.py` 의 `_DEFAULT_MODEL` 을 더 최신 무료 모델로 교체 (`ai.dev/rate-limit` 확인).

### Q. Supabase 에 접근 못 함
→ service_role 키 노출 가능성 → 즉시 Supabase 대시보드에서 Reset. `.env` 재발급.

### Q. assets/ 없어도 되나
→ Day 6 UI 작업엔 불필요. `GET /documents`·`/search` 가 기존 Supabase 데이터에 의존. Day 7 실 E2E 때 필요하면 그 때 복사.

### Q. 집 컴퓨터에서 하던 branch 랑 충돌
→ 양쪽 머신 다 `main` 한 브랜치. 새 머신 가기 전 이전 머신에서 `git push`, 새 머신에서 `git pull` 을 습관화. 충돌 예상되면 먼저 `git status` 확인.
