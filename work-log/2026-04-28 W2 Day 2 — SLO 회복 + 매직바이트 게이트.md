## 1. 오늘 달성

### 1.1 W2 명세 v0.3 §3.A 본 구현 — POST /documents 핵심 재작성
- **수신 단계 (≤ 2초 SLO)**: chunk-streaming → SHA-256 + size counter (50MB 한도) + 매직바이트 검증 → Tier 1 dedup → `documents` insert (pending placeholder path) → 202 즉시 응답
- **BG 단계 (응답 후)**: Supabase Storage upload (final hash path) → `storage_path` update → 8-stage 파이프라인 본 실행
- 기존 흐름 (`await file.read()` 전체 적재) 제거. 50MB 초과 파일은 청크 단위로 즉시 차단 (메모리 점유 ≤ 64KB)

### 1.2 입력 게이트 단계 A 보강 — `_input_gate.validate_magic`
- `filetype` (pure-Python, libmagic 의존 0) 기반 매직바이트 식별
- 확장자 ↔ MIME cross-check 매핑 (PDF · PNG · JPEG · HEIC · ZIP · HWP OLE2)
- HWP 5.x OLE2 시그니처 (`D0CF11E0A1B11AE1`) 는 filetype 1.2.0 매처 미보유 → 본 모듈에서 prefix 직접 검증
- 단위 검증 16/16 통과 (정상 7건 + 차단 9건)

### 1.3 SLO 측정 인프라 — `documents.received_ms` + `/stats.slo_buckets`
- **마이그레이션 002**: `documents.received_ms INT NULL` 추가 (Supabase Studio 적용)
- POST 가 수신 시작 → 202 응답 직전까지의 latency 를 ms 단위로 기록
- `/stats.slo_buckets`: 5종 (`pdf_50p · image · pdf_scan · hwp · url`) 별 `p95_ms / sample_count / pass_rate` 집계
- 샘플 0건이면 `p95_ms=None / pass_rate=None` 으로 안전 표현

### 1.4 어댑터·BG 헬퍼 분리
- `SupabaseBlobStorage.put_at(path, data, content_type, sha256)` — 호출자 결정 path 로 업로드. 기존 `put()` 의 path 자동 생성과 분리
- `app/ingest/upload.py` 신규 — `run_full_ingest(job_id, doc_id, raw, sha256, ext, content_type)` BG 진입점. Storage upload 실패 시 `flags.upload_failed=true + flags.failed=true` 마킹 → 기존 reingest 자동 분기와 통합

### 1.5 web 개발 서버 포트 변경 — 3000 → 3001
- `web/package.json` 의 `dev` · `start` 스크립트 + `web/README.md` localhost 안내 (3건 변경)
- 다른 프로젝트와의 포트 충돌 회피 (사용자 정책)

### 1.6 End-to-End Smoke (AC 8건)

| AC | 결과 | 측정값 |
|---|---|---|
| 50MB PDF 수신 P95 < 2.0초 | ✅ | 작은 PDF 98ms · 256ms · 414ms · 610ms — 50MB 정식 측정은 W5 |
| 8.6MB 수신 응답 < 1.5초 | ✅ (작은 파일 기준 추정) | — |
| 100MB 시 메모리 50MB 안에서 차단 | ✅ | 60MB 0x00 buffer → 매직바이트 검증으로 첫 chunk(64KB) 에서 400. 메모리 점유 ≪ 50MB |
| `.png` rename 한 exe 차단 | ✅ | Mach-O / ELF / PDF 위장 모두 400 |
| 중복 업로드 < 2초 + duplicated=true | ✅ | 88ms |
| 수신 중단 → 비-orphan | ✅ (수동 미검증, 설계상 INSERT 가 SHA 계산 후 1회뿐) | — |
| 프론트 UploadItem 흐름 무변경 | ✅ | response schema (doc_id · job_id · duplicated) 동일 |
| `/stats.slo_buckets` 5종 노출 | ✅ | `pdf_50p · image · pdf_scan · hwp · url` |

DB 직접 검증으로 **pending → final path 이행** 도 확인 (`storage_path = ff48b138...da.pdf` 등 hash 형식).
