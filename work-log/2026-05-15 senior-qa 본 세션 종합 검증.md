**재현**:
- `evals/golden_v2.csv` 현재 132 row (P4·P4-b 정리 후)

- 그러나 **PRD master v1.5 §변경 이력** 은 본 결과를 "BM25 IDF 분포 변화 우려 = 무근거 실증" 으로 그대로 채택

**문제**:
- 50 row 차이 = 분모 변동 38% (132/182) — 통계적으로 두 R@10 값은 직접 비교 불가
- 사라진 50 row 의 qtype 분포가 평균을 어떻게 끌고 갔는지 미분석
- caption_dependent gap 부호 반전 (+0.012 → −0.026) 도 같은 confound

**영향**:
- "회귀 신호 0" 결론을 v1.5 sprint 진입 PASS 신호로 채택했는데, 실제로 IDF 영향이 가장 큰 SK 사업보고서 (chunks 25806, 1432p) 가 golden_v2 에 0건 = 측정 대상 외
- 진정한 over-chunking 영향은 새 3 도메인 doc 의 자체 query 가 들어가야 측정 가능 (golden_v3 후보)

**권장 대응**:
- PRD v1.5 §변경 이력의 "회귀 신호 0" 표기를 "row 수 confound 큼, 새 doc query 부재 — qtype·분포 신호 기반 정성 평가" 로 보강
- 또는 sparse-only 측정 (`--mode sparse`) 1회 추가로 BM25 IDF 직접 영향 분리 측정

### P1-2. 영어 본문 list item false-positive (권고 5)

**재현 (직접 매치 테스트)**:
```
'1. The first item is foo'  → _HEADING_TEXT_PATTERN.match = True
'1. We propose a method'    → _HEADING_TEXT_PATTERN.match = True
'2. The next step'          → _HEADING_TEXT_PATTERN.match = True
```

**문제**:
- 패턴 `\d+(?:\.\d+){0,3}\.?\s+[A-Z][A-Za-z]*` 는 첫 단어가 대문자 시작이기만 하면 매치 → 영어 본문의 enumeration 첫 단어가 대문자면 false-positive
- 단 `_HEADING_TEXT_MAX_LEN=80` cap 이 본문 list 의 긴 문장은 차단 — work-log §회귀 위험 평가에 명시됨
- 그러나 list item 이 80자 미만이면 통과

**영향**:
- 한국어 자산 KPI 회귀 0 — 한국어 자산엔 `[A-Z]` 시작 없음 = 안전
- 영어 학술 PDF 본문에 짧은 enumeration list 가 있는 경우 잘못 heading 으로 잡혀 sticky propagate 시작점이 됨

**권장 대응**:
- arXiv reingest 후 section_title 분포 확인 시 enumeration list 가 heading 으로 잡혀 있는지 sample 확인
- 필요 시 영어 numbered 패턴에 trailing context 제약 (다음 세션 후속 작업 후보)
