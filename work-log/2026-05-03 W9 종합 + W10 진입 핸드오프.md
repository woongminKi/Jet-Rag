## 3. 두 단계 quota 보호 매트릭스 (W9 핵심 자산)

```
사용자 doc 인제스트 → 9 stage chain → Gemini Flash RPD 20 cap
                                      ↑                    ↑
                    Vision (스캔 PDF + PPTX)        tag_summarize (tags + summary)
                            ↓                                  ↓
              [cap=5 시도 기준]               [_call_tags 첫 호출]
                       ↓                                  ↓
                       ✗ quota?              quota 감지 → summary skip
                       → fast-fail (1회 stop)
                       
                       감지 정확도:
                       1) class name (ResourceExhausted, TooManyRequests)
                       2) status_code/code == 429
                       3) 메시지 키워드 (RESOURCE_EXHAUSTED / 429 / QUOTA)
```

### 3.1 효과 측정 (실제 reingest 비교)

| Stage | quota 시 호출 | W9 fix 후 |
|---|---:|---:|
| PptxParser Vision (11 slide PPTX) | 11 (cap 무력화) | **1** (90% 절약) |
| tag_summarize LLM | 2 (tags + summary) | **1** (summary skip, 50% 절약) |

### 3.2 부수 효과 — Vision 절약이 LLM 살림

W8 Day 2 reingest 의 stage 로그:
- `tag_summarize | failed | 429 RESOURCE_EXHAUSTED`

W9 Day 5 (fast-fail 적용) 의 reingest:
- `tag_summarize | succeeded` ← Vision 11→1 절약 → quota 여유

→ Gemini RPD 가 Vision + LLM 공유라 한 stage 절약이 다른 stage 보호.
