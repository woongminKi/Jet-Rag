## 4. M0-b 작업 후 → M0-c 영향

- G-U-018·G-U-027 처리 방식(query 재작성 vs 제거)에 따라 `n_chunk_evaluable` 가 176(재작성·라벨 측정 가능) 또는 174(2 row 제거) 또는 그 사이로 변동 → **M0-c baseline 재측정에서 확정**.

- 비대칭 3 row 의 라벨 변경(acceptable 축소 등) → cross_doc R@10 (현 0.4424) 미세 변동 가능 → M0-c 에서 재측정.
- ⚠️ **`evals/build_golden_v2.py` 재실행 주의**: golden_v2.csv 를 수동 편집한 뒤 `build_golden_v2.py` 를 다시 돌리면 cross_doc 9 row 는 early-return 으로 보존되지만 다른 변경분은 소실될 수 있음 (docstring 경고 참조). M0-b 편집은 직접 CSV 편집 + `.bak.YYYYMMDD` 백업 권고.

## 5. 작업 메모

- DB 조회는 read-only (`execute_sql` SELECT only) — DB 무변경. golden_v2.csv 무변경.
- chunk text 인용은 일부만 발췌(좌 200~500자) — 전체 확인은 M0-b 작업 시 `select text from chunks where doc_id=... and chunk_idx=...` 로.
- `caption_dependent` 류 vision chunk 존재 여부(G-U-005)는 미확인 — M0-b 에서 `select chunk_idx, section_title from chunks where doc_id='92c38315-...' and section_title ilike '%vision%' order by chunk_idx` 로 확인 권고.
- 직제규정 내 명시적 "위원회" chunk 존재 여부(G-U-015)도 미확인 — M0-b 에서 `select chunk_idx, section_title, left(text,200) from chunks where doc_id='a5f3cfae-...' and (text ilike '%위원회%' or section_title ilike '%위원회%')` 로.
