## 설계 결정 (비판적 재검토 적용)

### 1) 페이지마다 가드 vs 사전 1회 vs N=5 페이지마다

후보 비교 (3회 자기검토):

| 옵션 | DB 부하 | 폭주 차단 정확도 | latency |
|---|---|---|---|
| A. 사전 1회만 | 1 SQL/doc | 약함 (현재 doc 안 누적 무시) | 0 |
| B. 페이지마다 | 50 SQL/doc | 강함 | +0.x초 누적 |
| C. 사전 + N=5 | 11 SQL/doc | 균형 (cap 도달 후 5페이지 안 감지) | 0.x초 |
| D. 사전 + 추정값 | 1 SQL/doc | 균형 (페이지 수 × avg cost 사전 추정) | 0 |

**채택: C**.
- 이유: D 는 avg cost 측정이 D3 단계라 정확도 잠정. C 는 정확한 누적 cost 를 SQL 로 직접 보고, 50p PDF 90초 SLO 안에서 무리 없음.
- ENV `JETRAG_BUDGET_RECHECK_EVERY_N_PAGES` 로 운영자 조정 가능.

### 2) daily 정의 — UTC midnight vs 24h sliding

master plan §7.4 의 "24h 누적" 표현은 sliding 의도. 그러나 D4 본 ship 은 **calendar day (UTC midnight 리셋)** — 단순 + 가시성 (사용자 입장에서 "오늘 / 내일" 이 직관적). D5 에서 24h sliding 추가 예정 (별도 helper, 이중 검사로 호환).

### 3) DB graceful — allowed=True default

마이그 014 미적용 환경에서 SUM SQL 이 컬럼 부재로 fail 하면 두 옵션:
- allowed=False (보수적): 운영 graceful 깨짐 — 대량 인제스트 차단
- allowed=True (관대): cap 미적용 동작 (현 운영 그대로)

**채택: 관대**.
- 이유: vision_cache / vision_metrics 와 동일 패턴 — DB graceful default 가 일관성 있음. 마이그 014 적용 후 자동 회복.
- 첫 1회 warn 로그 → 운영자가 마이그 누락 인지 가능.

### 4) UI 카피 — "🚫 시각 보강 일부 생략 (월 한도 도달)" 후보 vs 본 채택

master plan task 가 제안한 카피는 "월 한도" 였으나 실제 구현은 doc / daily 로, 사용자에게 **"문서당 한도" / "일일 한도"** 가 더 정확. UI 카드는 scope 동적 라벨 적용:
- `scope='doc'` → "문서당 한도"
- `scope='daily'` → "일일 한도"

또한 누적/한도 USD 값을 카드에 표시 → 사용자가 "얼마나 더 기다리면 풀리는지" 판단 가능 (master plan §11.5 사용자 통제권 정합).

## 회귀 / 한계

- **race**: 같은 doc 의 동시 인제스트는 budget_guard 의 SUM 시점이 다를 수 있음. master plan default 채택 — best-effort, 단순 차단. 인제스트 동시성이 페르소나 A 에서는 1~2건이라 무시 가능.
- **cost 추정 정확성**: vision_usage_log.estimated_cost 가 NULL 이면 SUM 에서 제외 → 마이그 014 미적용 환경에서는 cap 가드가 동작 안 함 (DB graceful 의 의도된 결과).
- **재처리 시점**: 사용자가 "재처리" 클릭해도 cap 이 풀려있지 않으면 백엔드 가드가 graceful skip. UI 가 사용자 surprise 회피하려면 재처리 결과 폴링 시 다시 vision_budget_exceeded 가 true 로 돌아오면 안내 필요 — 본 ship 의 useEffect 폴러가 자연 처리. (UI 는 재처리 직후 "재처리가 시작되었어요" 표시 → 폴러가 latest_job 폴링 → flags 갱신 시 useEffect 가 재렌더 → 다시 안내).
- **D5 에서 보강할 항목**: 24h sliding window (calendar day 외 옵션) + Google AI Studio Usage 대시보드 cross-check 가이드.
