## 3. 사용자가 배운 것 (오늘 핵심 통찰)

### 3.1 "매칭 강도 100%" 의 진짜 의미
- 코드상 정상 (top-1 정규화 정의)
- 사용자 멘탈 모델 vs 시스템 의미의 간극
- **Ragas 측정으로 검증**: 검색은 정답을 찾고 있지만 ranking 격차 27% 존재 (Recall 100% vs Precision 73%)

### 3.2 무료 모델 → 유료 모델 전환의 trade-off
- BGE-M3 (다국어 SOTA) 가 한국어 자료에서 OpenAI 대비 우위 가능
- "OpenAI = 자동 향상" 은 흔한 오해
- **검색 품질의 본질 = 모델 < 데이터·정책** (D8 시도가 직접 증명: 모델 교체 없이 청크 정책 fix 가 ROI 높음)
- LLM 답변 = 새 기능 추가 ≠ retrieval 향상

### 3.3 Ragas 의 진짜 가치
- 모델·정책 비교의 ground truth 도구
- D8 1차 시도 실패를 즉시 정량 캐치 — 가드 ship 후 모르고 운영했을 위험 회피
- Phase 2 차수 결정의 정량 신호 제공

### 3.4 Jet-Rag 어댑터 레이어 narrative
- 동일 데이터셋 + 같은 검색 RPC + 다른 모델 = 정량 비교 인프라 ship 완료
- `impl/__init__.py` default factory 1줄 변경으로 모델 swap 가능
