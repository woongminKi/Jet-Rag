### 6.2 페르소나 "빠르게" vs latency
- W-1(a) decomposition: cross_doc query 당 +1~2초. cross_doc 은 전체 query 의 ~5% → 전체 p95 무영향. simple query 는 게이트 통과 안 함 → latency 무변동. 단 decomposition latency 가 cross_doc query p95 를 3초 초과시키면 게이트 (M1 DoD).
- W-4 클린 재인제스트: 인제스트 시간 증가 (chunk augment + HF cold-start). 인제스트는 비동기 워커 (사용자 대기 < 2초 = 접수) → 체감 영향 적음. 인제스트 SLO 달성률 ≥ 90% (기획서 #11) 게이트 — 분할 재인제스트로 cold-start stall 회피.
- W-9 paid 답변: 사용자 클릭 시만 → 자동 latency 없음.

### 6.3 paid 비용 폭주 방어 (plan 원칙 5개 유지)
- ① 검색 기본 경로 무료 — W-1(a) decomposition 은 gated, 기본 검색 $0.
