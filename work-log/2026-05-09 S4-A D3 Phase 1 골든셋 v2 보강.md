### 2.1 doc 매칭 자동 fix

| 항목 | 값 |

**핵심 fix 사례 — `(붙임2)` prefix 데코 흡수**

- v1 표기: `2025년 데이터센터 산업 활성화 지원 사업 통합_안내서`
- DB title: `(붙임2) 2025년 데이터센터 산업 활성화 지원 사업 통합_안내서 (1)`
- 매칭 path 3 (substring 양방향 + 데코 strip) 으로 흡수.
- G-U-006~009 4 row 모두 doc_id 채움.

**hidden bug — macOS NFD 인코딩**

- Supabase 에서 가져온 title 의 한글이 **NFD (조합형)** 으로 저장 (macOS HFS+ 영향).
- regex `[가-힣]*` 가 NFD 자모 분리 문자에는 매칭 실패 → `_strip_decoration` 이 `(붙임2)` 를 못 떼어냄.
- fix: `_strip_decoration` 에서 NFC 정규화 선행 (`unicodedata.normalize("NFC", text)`).
- 단위 테스트는 직접 작성한 NFC string 으로 검증되어 통과했지만 본 PC 실 데이터 실행 시 발견 → 단위 테스트와 실 환경 회귀를 둘 다 돌리는 가드 정당성 재확인.

## 남은 이슈

### Q-S4-A-D3-1 — relevant 자동 채움 수가 적음 (27건)

v1 row 157 중 relevant_chunks 가 비어 있던 row 만 자동 추출 대상. 대다수 v1 row 는 이미 채워진 상태라 27건은 정상 범위. 다만 v1 의 G-U-* 사용자 row 32건 중 doc_id 가 비어 있던 row 와 G-N-* negative 5건 제외하면 자동 채움 후보가 한정적. v2 자체의 row 추가 (Phase 2/3) 가 본질적 보강.

### Q-S4-A-D3-2 — caption_dependent 룰의 chunk metadata 부재

본 PC chunks.metadata 에 `table_caption` / `figure_caption` 키가 없어 chunk text 패턴 (표/그림/도식/Figure/Table/다이어그램/도표) 으로 대체. 향후 인제스트 파이프라인 (S2 vision enrichment) 이 metadata 에 caption 키를 채워주면 정확도 상승. 현재 룰은 false positive 위험 — 일반 본문에 "표" 단어가 있으면 마킹됨.

### Q-S4-A-D3-3 — BGE-M3 HF API 503 / timeout 빈도

build 1회 실행 중 7~8회 timeout 발생 (`The read operation timed out`). bgem3_hf_embedding 의 retry 3회로 자동 복구되지만 G-U-005 / G-U-007 은 모든 retry 실패 → query embed skip → relevant 채움 누락. 현재 fail-safe 로 빈 relevant 보존했지만 향후 HF inference endpoint 안정화 또는 fallback (해당 row 만 v1 의 빈 값 그대로) 정책 필요.

### Phase 2 / Phase 3 Deferred

- **Phase 2 — cross-doc 보강**: 현재 cross_doc 4건이 모두 v1 사용자 작성 row (G-U-015/017/018/026/027). 자동 추출은 sub_doc 별 top-1 합산만 적용. cross_doc 비교 query 의 query 자체 다양화는 Phase 2 에서 LLM 활용 (Gemini 호출 발생) — 명세에서 진입 X.
- **Phase 3 — 다른 PC 자료 통합**: 어제 핸드오프 §3 의 "문제 PDF 는 다른 컴퓨터" 이슈와 연동. 본 PC docs 13건만 대상. 다른 PC 의 자료를 v2 에 통합하려면 Supabase storage 동기화 또는 별도 fixture 정의 필요 — Phase 3 진입 X.
