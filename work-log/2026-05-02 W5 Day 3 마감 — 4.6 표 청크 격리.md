## 3. 핵심 변경

### 3.1 `_looks_like_table_cell` 헬퍼

```python
def _looks_like_table_cell(text: str) -> bool:
    if " | " in text:  # DocxParser/Markdown 표 텍스트
        return True
    stripped = text.strip()
    if len(stripped) >= 30:
        return False
    non_ws = sum(1 for c in stripped if not c.isspace())
    if non_ws == 0:
        return False
    digit_punct = sum(
        1 for c in stripped
        if c.isdigit() or (not c.isalnum() and not c.isspace())
    )
    return digit_punct / non_ws >= 0.5
```

## 6. 비판적 자가 검토

1. **4.9 (chunk_idx 추적 메타) 이월 사유**: split + merge 후 original idx 추적은 ExtractedSection 메타 추가 필요. 디버깅 가시성 가치 낮음 (현재 검색 결과는 chunk_id 로 충분 추적 가능). W6+ 정량 측정 시 chunks 분포 분석에 필요할 때 ship.
2. **표 셀 판정 임계 (30자, 50%)**: chunk_filter 의 table_noise (90%, 70%) 보다 loose. 분리는 보수적으로 — false negative (표 셀이 본문으로 분류) 보다 false positive (본문이 표 셀로 오판) risk 우선. 페르소나 A 자료 누적 후 임계 calibration.
3. **DOCX 표 처리 정합**: DocxParser 의 ` | ` separator 가 즉시 감지됨 → DOCX 표 청크는 자동으로 분리 + chunk_filter table_noise 마킹 가능. 기획 §10.5 정합.
4. **dry-run worst-case 변화 미미 (-0.2pp)**: 기존 chunks 분포에서 표 셀 패턴 보유 청크가 적음. 실 효과는 신규 DOCX/PDF 인제스트 시 측정. 인프라 사전 ship.
5. **회귀 0 검증**: `test_normal_short_sections_still_merged` 가 일반 본문의 기존 병합 동작 보존 검증.
