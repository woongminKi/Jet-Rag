## 6. C0 baseline (37.4% skip) 회귀 0 확인

- D4 baseline 측정값: skip 37.4% (D3 CSV 기준)
- v3 적용 후 (CSV regen): skip 12.8% — 데이터센터 안내서 p.1~41 의 table_like 회수가
  핵심 원인. 즉 "skip rate 회귀" 가 아니라 **"회수 효과로 인한 skip rate 감소"** 임
- false positive 측면: A4_quality_only / A5_caption_only 의 skip rate 100% / 97% 은
  C0 와 무관 (단독 ablation). C0 는 100.0% hit_rate + DC p.40 catch=Yes 유지.

**C0 baseline 12.8% 의 의미** (P1-3 명시): D4 ablation 의 C0 baseline (37.4%) 와
동일 임계 (1e-3 / 0.30 / 0.30 / 0.40 / 0.20) 이지만, **입력 CSV 가 v3 산출 모듈로
regen** 되어 데이터센터 41 pages 의 `table_like_score` 가 0 → 0.99 로 상승한 결과.
즉 ablation 비교는 **CSV regen 동등 환경에서만 의미** — D4 의 37.4% 와 v3 의 12.8%
는 직접 비교 불가. 임계 변경 효과를 보려면 같은 CSV 위에서 후보별 결정값을 비교
(C0 12.8% vs C2 4.6% 등) 해야 한다.
