## 3. 핵심 발견 (다음 세션이 알아야 할 사실)

### 3.1 search-side cross_doc/synonym fix 는 막다른 골목

- W-1(a) paid decompose: cross_doc R@10 0.4424→0.4216 (net-neg).
- W-1(b) 옵션 A doc-scoped 필터: cross_doc R@10 0.4424→0.4080 (또 net-neg).

- 원인: 라벨 doc 의 baseline chunk-RRF 자체가 약함. search-side 의 후속 재배치만으로는 해결 불가. → M2 chunk augment 가 진짜 fix 였음.

### 3.2 W-3 caption prefix 가 M2 의 진짜 성과

- table_lookup top-1 ~0.5 → 0.9167 (사실상 1.0)
- caption_dependent gap +0.28 → +0.012

### 3.3 인제스트 비결정성 noise floor

- 같은 코드·같은 doc 으로 reingest 해도 top-1 ±0.005~0.012 변동 (1~2 row in 177).
- 원인: tag_summarize Flash-Lite LLM temperature·BGE-M3 batch 순서·entity_extract 등.
- → top-1 0.78~0.80 구간은 통계적 잡음 안의 soft threshold. M2 W-4 final 0.7966 = 0.80 의 -0.0034pp 는 noise band 내 → "사실상 0.80 도달" 가능 (보수적 보고 시).
- 추가 surgical 실험 비생산적 — wandering 만 야기 (1 UP / 3 DOWN 패턴 반복).

### 3.4 vision_budget 코드 버그 (P1)

- `documents.flags.vision_budget.used_usd` 가 historical 누적 — vision_page_cache hit 도 used 에 charge (잠재 버그).
