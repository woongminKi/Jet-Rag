`config._DOC_BUDGET_USD_DEFAULT = 0.10` 의 산정 근거:

doc 다양성 부족 (n=2) 으로 21.5p 평균이 페르소나 A 의 실제 분포를 대표 못 함 → 측정값 사용 보류. 1주 추가 누적 후 재산정.

## 4. 설계 결정

### 4.1 services + scripts 분리 패턴

기존 `app/services/vision_metrics.py` + `scripts/monitor_search_slo.py` 패턴 그대로:

- 계산 로직은 services (단위 테스트 재사용 + 향후 endpoint 도입 가능)
- CLI 는 scripts thin wrapper (DB I/O + argparse)

### 4.2 잠정/측정 임계

`_MIN_SAMPLE_ROWS=30` + `_MIN_UNIQUE_DOCS=5` — master plan §6 D3 함의 (1주 + 5 doc 평균) 그대로. 둘 중 하나라도 미달이면 잠정값 fallback.

### 4.3 NULL row 처리

| case | 처리 | 근거 |
|---|---|---|
| `success=False` | failed_rows 만 증가, sample 제외 | 실패 cost 가 평균 왜곡 |
| `estimated_cost NULL` | sample_rows 제외 | 단가 평균에서만 제외, doc 카운트엔 포함 |
| `doc_id NULL` | unique_docs 분리 + null_doc_rows 별도 카운트 | image_parser 단독 호출 / doc 삭제 SET NULL row — PDF 인제스트 budget 분석 외 |

### 4.4 default fallback 값

config.Settings 의 default 와 scripts/compute_budget.py 의 fallback 이 정확히 일치 — 운영 일관성. doc_budget=0.10 / daily=0.50 / krw=1380.

### 4.5 exit code

- 측정값 (충분 sample) → 0
- 잠정값 (누적 부족) → 1

CI gate 호환 — 추후 weekly cron 으로 이 스크립트 돌려 측정값 진입 시점 자동 감지 가능 (S0 종료 후 후속).
