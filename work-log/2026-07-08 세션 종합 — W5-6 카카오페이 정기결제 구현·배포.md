## 3. 2단계 리뷰가 잡은 실제 버그 (전부 수정 완료)

플랜 코드에도 결함이 있었고, 리뷰가 잡았다:

- 🔴 **CRITICAL — 재구독 다운그레이드**: `start_subscription` 이 무조건 `status="canceled"` upsert → 기존 active 구독자가 재클릭하면 Pro 즉시 박탈 + 재청구도 안 됨. → 기존 행은 `pending_tid` 만 갱신, 신규만 placeholder insert (`5def006`).
- 🟠 **배치 유저 격리**: per-user DB write 를 try 로 감싸 1건 실패가 전체 중단 안 되게 (`5def006`).
- 🟠 **이중청구 멱등 마커**: 결제 성공 후 기간 갱신 실패 시 다음 배치 재청구 방지 — `payment_history` charge_success 마커(period_key) (`5def006`).
- 🟠 **billing_key 없음 무한 skip → past_due 처리** (`5def006`).
- 🟠 **KakaoPay 응답 파싱 방어**: 비-JSON/비-dict 2xx 를 PaymentError 로 (배치 루프 보호) + 에러경로 테스트 (`7664ad6`).
- 🟡 **해지 확인 문구**: "다음 결제일부터" → "즉시 Free 전환"(실제 동작 일치) (`d710048`).
- 🟡 **/billing/success 새로고침 거짓 에러**: 409 시 /me/subscription 재확인 후 성공 표시 (`6f95a80`).
- 🟡 **/billing/run 부분 오설정 500 → 503** (`5fc099e`).

## 5. 남은 작업 · 진행 재개 지점

### 5.1 카카오페이 심사 통과 후 (활성화 절차)
1. **SECRET_KEY 추가** — 콘솔에서 발급받은 `JETRAG_KAKAOPAY_SECRET_KEY` 를 Railway 에 추가 → Deploy. (넣는 순간 결제 활성화.)
2. **production CID 교체** — `JETRAG_KAKAOPAY_CID` 를 TCSUBSCRIP → 전용 CID.
3. **Railway cron** — `0 18 * * *`(UTC=KST 새벽3시) `cd api && uv run python scripts/billing_charge.py`. (SECRET_KEY 넣기 전엔 켜지 말 것 — provider 생성 시 RuntimeError, 구독자 0이라 무해하나 로그 노이즈.)
4. **약관 본문** — `/terms`·`/privacy` 의 `TERMS_BODY`/`PRIVACY_BODY` scaffold 를 실제 초안으로 교체.
5. **sandbox e2e 테스트** — 구독 → KakaoPay 결제창 → `/billing/success` approve → `/me/subscription` active 확인 → 이메일 인제스트 Pro 게이트 통과 확인 → 해지 → canceled 확인.

### 5.2 W7 fast-follow (코드리뷰 defer 항목)
- **이중청구 하드닝** — 결제성공 후 멱등마커 write + 기간갱신이 **둘 다** 실패하는 narrow window 는 `payment_history(user_id,event,detail)` UNIQUE 제약 + attempt/success 이중 레코드로 exactly-once 강화 권장. 현재는 period-key 마커로 대부분 커버(beta 허용 수준, 코드에 "수동 확인" 로그).
- **해지 정책** — 현재 **즉시** Free 강등(active 해지 시 잔여 유료기간 손실). "기간말 해지 유예"(cancel_at_period_end)가 일반적 SaaS 패턴 — schema+배치+quota 변경 필요, **제품 결정 사항**.
- **가격 상수** — KakaoPay 어댑터 `_TOTAL_AMOUNT=6900` 하드코딩, `plans.price_krw` 와 수동 동기. 가격 변경 시 파라미터화 검토.
- **배치 pagination** — select 에 PostgREST ~1000 cap, 유료유저 소수라 무영향(스케일 시 `.range()`).
- **stale plan_code** — cancel/sweep 이 status=canceled 만 세팅, plan_code='pro' 잔존(무해 — entitlement 은 status 기반).

### 5.3 미검증 주의점
- `TCSUBSCRIP` 은 **구 API**(kapi.kakao.com, Admin Key) 샌드박스 CID 로 알려진 값. 코드는 **신 open-api**(open-api.kakaopay.com, SECRET_KEY 방식) 사용. 신 open-api 의 샌드박스 CID/테스트키 발급 방식은 카카오페이 콘솔/최신 문서로 확인 필요(미검증).
