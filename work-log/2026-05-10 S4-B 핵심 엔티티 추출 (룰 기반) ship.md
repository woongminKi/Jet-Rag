**API**:

```python
result: ExtractedEntities = extract_entities(text)
# result.dates = ["2024년 4월 30일"]
# result.amounts = ["50,000원"]
# result.percentages = ["50%"]
# result.identifiers = ["2288-7083"]
# result.to_dict() / result.is_empty()
```

**개별 함수**: `extract_dates(text)`, `extract_amounts(text)`, `extract_percentages(text)`, `extract_identifiers(text)` — 모두 List[str] 반환, dedup 적용.

### 1.2 정규식 패턴

**날짜 (3 패턴)**:
- `(\d{4})\s*년\s*(\d{1,2})\s*월(?:\s*(\d{1,2})\s*일)?` — 한국어 (Korean char 가 \w 인 점 고려, `\b` 제거 + greedy)
- `(?<!\d)(\d{4})[.\-/]\s*(\d{1,2})[.\-/]\s*(\d{1,2})(?!\d)` — ISO
- `(?<!\d)(\d{4})[.\-]\s*(\d{1,2})(?!\d)` — 월까지만

**금액 (4 패턴)**:
- `\d{1,3}(?:,\d{3})+(?:\.\d+)?\s*원` — comma 포함 원
- `\d{1,4}(?:\.\d+)?\s*(?:만|억|조|천)\s*원` — 한국 단위
- `\$\d{1,3}(?:,\d{3})*(?:\.\d+)?` — 달러
- `₩\d{1,3}(?:,\d{3})*(?:\.\d+)?` — 원화 기호

**백분율**: `\d{1,3}(?:\.\d+)?\s*%`

**식별자 (3 패턴)**:
- `ISSN\s+(\d{4}-\d{3}[\dXx])`
- `ISBN\s+([\d\-Xx]{10,17})`
- `제\s*\d+\s*호` — 한국 법령번호

### 1.3 단위 테스트 (16 신규)

- `ExtractDatesTest` (5): 한국어 full / 한국어 월만 / ISO 다양 / dedup / 빈 text
- `ExtractAmountsTest` (4): comma / 한국 단위 / 달러 / no-match
- `ExtractPercentagesTest` (2): integer / decimal
- `ExtractIdentifiersTest` (2): ISSN / 한국 법령번호
- `ExtractEntitiesTest` (3): integrated + to_dict + is_empty

### 1.4 한국어 regex 함정 (실측 학습)

초기 패턴 `\b(\d{4})\s*년\s*(\d{1,2})\s*월(?:\s*(\d{1,2})\s*일)?\b` 의 trailing `\b` 가 "30일부터" 같이 한국어 인접 시 매칭 실패 (Korean `일` + `부` 모두 `\w` → no boundary). 결과: "2024년 4월 30일" 대신 "2024년 4월" 만 매칭. **Fix**: `\b` 제거 + greedy match + ISO 패턴은 `(?<!\d)` / `(?!\d)` lookbehind/lookahead 사용 → 16 tests 통과.
