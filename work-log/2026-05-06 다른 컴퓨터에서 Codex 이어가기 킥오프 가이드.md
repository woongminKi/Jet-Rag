## 1. 사전 점검 5 단계

| # | 작업 | 명령어 / 절차 |
|---|---|---|
| 1 | repo 동기화 | `git clone https://github.com/woongminKi/Jet-Rag.git && cd Jet-Rag` 또는 `git pull origin main` |
| 2 | 의존성 설치 | `cd api && uv sync && cd ../web && pnpm install` |
| 3 | `.env` 복원 | 1Password / iCloud Keychain 등에서 본 컴퓨터의 `api/.env` + `web/.env` 값 복사 (짝 문서 §2·§3 참조) |
| 4 | `.codex/` 새 init | `cd Jet-Rag && codex init` (또는 codex CLI 의 setup 절차) — 본 컴퓨터 `.codex/` 는 `.gitignore` 처리되어 git 에 없음 |
| 5 | Supabase Personal Access Token 새로 발급 | https://supabase.com/dashboard/account/tokens → `.codex/config.toml` 의 `SUPABASE_ACCESS_TOKEN` 에 입력. **구 토큰 revoke 권고 (회전)** |

### 4.1 S0 D2 — vision_page_cache (마이그 015)

```text
S0 D1 마이그 014 완료. 이어서 S0 D2 진행.

작업: `vision_page_cache` 마이그레이션 015 신규.
- 컬럼: id (UUID PK), sha256 (TEXT), page (INT), prompt_version (TEXT),
  result (JSONB), estimated_cost (NUMERIC), created_at (TIMESTAMPTZ)
- UNIQUE 제약: (sha256, page, prompt_version)
- 인덱스: (sha256, page) 조회 최적화

+ extract.py 의 `_enrich_pdf_with_vision` 에 lookup → 결과 저장 로직 추가
+ 단위 테스트: 같은 sha256+page+prompt_version 으로 두 번 호출 시
  두 번째 호출은 cache hit (Gemini 호출 0)

master plan §7.2 SQL 스니펫 + §7.3 흐름 참고.
```

### 4.2 S0 D3 — sweep×retry 곱셈 제거

```text
S0 D2 완료. S0 D3 진행.

작업: vision worst case 호출 9 → 2 로 축소.
- api/app/ingest/stages/extract.py: _VISION_ENRICH_MAX_SWEEPS 3 → 2
- api/app/adapters/impl/_gemini_common.py: _MAX_ATTEMPTS 3 → 1
- ENV 화: JETRAG_VISION_MAX_SWEEPS, JETRAG_GEMINI_RETRY (default 위 값)
- 기존 단위 테스트 갱신 + 회귀 0 확인

근거: 503 random 은 1회 retry 면 충분, sweep 2 가 페이지 단위 재시도 보장.
master plan §7.3 참고.
```

### 4.3 S0 D4 — cost budget cap + graceful degrade

```text
S0 D3 완료. S0 D4 진행.

작업: 문서당·일별 cost cap 메커니즘.
- api/app/services/budget_guard.py 신규
  - check_vision_budget(doc_id) → BudgetStatus.OK / DOC_EXCEEDED / DAILY_EXCEEDED
- extract.py 가 vision 호출 직전 check_vision_budget 호출
- cap 도달 시 graceful degrade:
  - 무료 파서 결과 정상 저장 (vision 만 skip)
  - documents.flags.vision_budget_exceeded = true
  - UI 노출용 flag
- 사용자 재처리 endpoint (이미 있음) 와 호환

ENV: JETRAG_DOC_BUDGET_KRW, JETRAG_DAILY_BUDGET_KRW
초기값은 S0 D5 데이터 기반 측정 후 결정.
```

### 4.4 S0 D5 — budget 초기값 데이터 기반 설정

```text
S0 D4 완료. S0 D5 진행.

작업: 1주 누적 usage_metadata 기반 budget 초기값 산출.
- 평균 단가: SELECT AVG(estimated_cost) FROM vision_usage_log
  WHERE source_type='pdf_vision_enrich'
- 사용자 PDF 평균 페이지 수 측정
- DOC_BUDGET = 평균단가 × 평균페이지 × 0.5 × 1.5
- DAILY_BUDGET = DOC_BUDGET × 평균 인제스트 doc/일

산출 결과를 ENV 에 입력 + work-log 작성.
master plan §7.5 참고.
```


