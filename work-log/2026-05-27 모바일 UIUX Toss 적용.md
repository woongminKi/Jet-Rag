## 2. Toss 디자인 언어 적용 매트릭스

| 원칙 | 적용 결과 |
|---|---|
| 카드 border-radius | shadcn Card 기본 `rounded-xl` 위에 `rounded-2xl` className override — Toss 풍 16px |
| 카드 padding | mobile `p-4` / desktop `md:p-6` — 일관 적용 |
| 8px 그리드 | gap-2/3/4 통일, 어색한 5/7px 제거 |
| 터치 타깃 | 모바일 h-11 (44px) 검색 input, h-9 (36px) 보조 버튼은 좌우 padding 으로 터치 영역 보강 |
| 폰트 | 본문 `text-base` (16px) — mobile 에서 iOS auto-zoom 방지. 메타 `text-xs`/`text-sm` 만 |
| 가로 스크롤 금지 | `body { overflow-x: hidden }` 안전망 + 컨텐츠는 모두 break-words/wrap 변경. 의도된 chip row 만 `scrollbar-hide` utility 로 허용 |
| safe-area | 헤더 `padding-top: env(safe-area-inset-top)` |
| sticky 그림자 | 헤더 + search subheader + ask header 에 `shadow-sm` 추가 |
| 색감 | 변경 없음 — 기존 oklch primary(남청) + accent(시안) 유지. Toss 와 다른 톤이지만 일관성 보존 우선 |

## 3. 사이드 이펙트 점검

- **Server Component first fetch 패턴**: 모두 유지 (`/` `/search` `/docs` `/ask` 의 page.tsx 는 RSC, AnswerView/MetricsTrendCard 등 client-only refetch 패턴 무변경)
- **React 19 lint rule** (`react-hooks/set-state-in-effect`): 본 변경은 JSX 클래스/구조만, hook 동작 무변경
- **useTransition + interruptible navigation**: search-subheader 의 mode 토글 동작 동일 — JSX 만 재배치
- **SVG 직접 그리기**: chunks-stats donut / metrics-trend sparkline 모두 viewBox + preserveAspectRatio 유지 → 카드 폭 변경에도 자동 fit
- **mobile-first responsive grid**: 기존 패턴 강화 (mobile px-4 → desktop md:px-6 일관)

**기능 회귀 가능성**:
- 0건 — 모든 변경이 className 및 layout 구조 변경. 데이터 흐름·API 호출·state 변경 없음
- ablation mode 토글 (hybrid/dense/sparse) 동작 동일
- citation [N] 클릭 highlight 동일
- doc 스코프 검색 / 매칭 청크 fetch 동일
- RAGAS judge 호출 동일

## 5. 미해결 이슈 / 후속 sprint 후보

### 5.1 본 sprint 의식적 deferred
- **다크 모드**: globals.css 에 dark 토큰은 정의돼 있지만 자동 전환 트리거 없음 — 시스템 prefers-color-scheme 연동 별도 sprint
- **landscape orientation**: 폰 가로 회전 시 검색 sub-row 의 chips 가 한 줄에 모두 들어가긴 함. 더 깔끔한 처리는 별도 sprint
- **Toss 색감 적용**: 사용자 요청은 "Toss 디자인 언어 기준" 이지만 현재 primary (oklch 0.45 0.15 250 — 남청)는 Karrot 풍에 가까움. Toss 정통 블루 (#3182f6) 매칭은 oklch 환산 후 토큰 swap 별도 sprint
- **카드 그림자 강도**: 본 sprint 는 `shadow-sm` 만 추가 (기존 카드는 Card primitive 가 `shadow-sm` 내장). Toss 는 hover 시 `shadow-md` 부드러운 lift 가 표준 — 시각 polish 별도 sprint

### 5.2 잠재 이슈 (확률 낮음)
- **shadcn Card primitive override**: 카드별 `className="rounded-2xl"` 로 `rounded-xl` 을 덮어쓰는데, primitive 자체 수정이 더 일관성 있음. 다만 admin/ingest 같은 기존 페이지에 사이드 이펙트 위험 있어 보류
- **`overflow-hidden` 카드**: 카드에 dropdown / tooltip 띄울 경우 잘릴 수 있음. 본 카드들은 popover 없음으로 확인했지만 향후 추가 시 주의
- **`break-words` vs `break-keep`**: 한국어 단어 중간 자르기 회피하려면 `break-keep` + `word-wrap: break-word` 가 더 자연스러움. 본 sprint 는 `break-words` (≈ word-break: break-word) 통일. 한국어 가독성 미세 polish 는 후속

- mobile 회귀 자동 측정 인프라 0 — Chromatic / Percy / playwright visual diff 도입 별도 sprint
